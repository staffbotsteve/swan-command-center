Example flow — "@research audit Q2 hospitality performance across NV + CA"

1 · Ingress
Telegram webhook → auth checks chat ID → queue accepts → router sees @research prefix → task created with projects=[Hosp NV Q2, Hosp CA Q2], companies=[Hosp NV, Hosp CA].
2 · Agent work + spawn
Research reads both company folders from vault. Calls NotebookLM to summarize prior Q-over-Q data. Spawns an ephemeral P&L-Analyst sub-agent (logged in spawn_log) to crunch numbers in parallel.
3 · Memory + registry
Insights land in Supabase. Hive-mind logs task completion. Sub-agent's 3rd successful run triggers promotion candidate in agent_registry — surfaced on dashboard for you to approve.
4 · Response + durable state
Reply dispatched back via Telegram. Weekly cron promotes tagged insights → 02-Areas/Research/2026-Q2-hospitality.md. Session summary written per vault-write rule.
Non-goals. No multi-tenant access. No OpenClaw / Paperclip / third-party CC harnesses. No per-company agent silos. No local Claude Code SDK runtime (uses Managed Agents API). No Pika avatars. No SMS / iMessage (abandoned for Telegram stability and cost). No agent-authored code running outside Vercel preview sandboxes without your PR approval.
